### Задача
* Разработать простейший сервис, предоставляющий RESTFul API для работы с
пользователями.

### Условия
 - Python версии > 3.6
 - Микро-фреймворк Bottle или на свой выбор
 - Библиотека для десериализации — marshmallow или любая другая на свой выбор

### Описание
* Сервис API должен:
    * хранить данные о пользователях и уметь манипулировать этими данными
    * создавать, возвращать и удалять информацию. 

Он должен поддерживать следующие методы:
    1. POST /user — принимает входящие данные в виде JSON-документа с полями "name" и
"age". Оба поля обязательны. Валидация: имя должно быть от 3 до 30 символов,
возраст от 0 до 99. В случае успеха, ответ должен быть 201 и содержать объект,
описывающий пользователя — это поля id, name и age. В случае ошибки, возвращать
код 400 и описание ошибки.
    2. GET /user/XXX , где XXX — ID пользователя. Выдает код 200 и информацию о
пользователе, если пользователь с таким ID существует, или ошибку 404, если не
существует.
    3. DELETE /user/XXX - удаляет пользователя, возвращает 204 (без контента) или отдает
ошибку 404, если пользователя не существует

* ID пользователей пусть будет строковое представление уникального случайного
идентификатора UUID (то есть ID вида 62ff2d6175a9214d5914ffae89cd7e48 ).
* Базы данных никакой не предусмотрено. Информацию о пользователях сервис
просто хранит у себя до момента его остановки.
